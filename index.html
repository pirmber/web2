Detailed Topic Descriptions

Data

Data refers to raw facts and figures, such as numbers, characters, images, or any other output that can be processed by a computer. It has no meaning on its own until processed and interpreted.

Information

Information is processed data that is meaningful and useful for decision-making. It is data that has been organized, structured, or presented in a context that gives it meaning.

Features of Information

  • Accuracy: Information must be correct and reliable.
  • Timeliness: Information should be available when needed.
  • Relevance: It must relate to the purpose at hand.
  • Completeness: Should contain all necessary details.
  • Consistency: Information should be consistent across sources.

Database and Its Purpose

A database is an organized collection of related data. Its purpose is to store, manage, retrieve, and update large amounts of data efficiently.

Terminologies in Database

  • Table: A collection of records in rows and columns.
  • Field: A column in a table; a specific attribute.
  • Record: A row in a table; a single data item.
  • Tuple: Another term for a record.
  • Object: A data entity in object-oriented databases.
  • Key: A field or set of fields used to identify records uniquely.

Data Dictionary

A data dictionary is a centralized repository of metadata, describing the structure, relationships, and constraints of data stored in the database.

Database Management System

Introduction

DBMS is software for creating, managing, and interacting with databases. It provides an interface between applications and the data.

Objectives

  • Provide efficient data storage and retrieval.
  • Ensure data integrity and security.
  • Support concurrent access.

Advantages

  • Data independence
  • Reduced data redundancy
  • Improved data sharing

Disadvantages

  • Complexity and cost
  • Performance overhead

Types of Database Model

  • Hierarchical: Tree-like structure with parent-child relationships.
  • Network: Graph-based structure allowing many-to-many relationships.
  • Relational: Data represented in tables, linked by keys.
  • Entity-Relationship: Conceptual model depicting entities and relationships.

Integrity Constraints and Types

  • Domain Constraint: Valid values for a given attribute.
  • Entity Integrity: Primary key cannot be null.
  • Referential Integrity: Foreign key must match a primary key in another table.
  • Key Constraints: Uniqueness of keys.

Normalization

Introduction

Normalization is the process of organizing data to minimize redundancy and dependency.

Normal Forms

  • 1NF: Eliminate repeating groups.
  • 2NF: Remove partial dependencies.
  • 3NF: Remove transitive dependencies.

Advantages

  • Reduces redundancy
  • Increases data integrity

Disadvantages

  • Can be complex
  • May require more joins

Centralized and Distributed Database

Introduction

Centralized DB stores data at a single location; distributed DB stores data across multiple sites.

Advantages

  • Centralized: Easier control.
  • Distributed: Fault tolerance, local autonomy.

Disadvantages

  • Centralized: Single point of failure.
  • Distributed: Complex management.

Comparison

Distribution improves availability, but adds complexity in synchronization and management.

Database Security

Introduction

Database security protects data from unauthorized access and threats.

Challenges

  • SQL injection
  • Unauthorized access
  • Data breaches

Security Measures

  • Authentication and authorization
  • Encryption
  • Auditing and monitoring

Roles of DBA

  • Backup and recovery management
  • User management
  • Performance tuning
